# Segurança

O GitHub leva a segurança de nossos produtos e serviços de software a sério, incluindo todos os repositórios de código aberto (open source) gerenciados por meio de nossas organizações no GitHub, como [GitHub](https://github.com/GitHub).

Mesmo que [repositórios open source estejam fora do escopo do nosso programa de recompensa por vulnerabilidades (bug bounty)](https://bounty.github.com/index.html#scope) e, portanto, não sejam elegíveis a recompensas, garantiremos que sua descoberta seja encaminhada aos mantenedores apropriados para correção. 

## Relatando problemas de segurança

Se você acredita ter encontrado uma vulnerabilidade de segurança em qualquer repositório pertencente ao GitHub, informe-a a nós por meio de divulgação coordenada.

**Não relate vulnerabilidades de segurança por meio de issues, discussões ou pull requests públicos no GitHub.**

Em vez disso, envie um email para opensource-security[@]github.com.

Inclua o máximo possível das informações listadas abaixo para nos ajudar a entender e resolver melhor o problema:

  * O tipo de problema (por exemplo, buffer overflow, SQL injection ou cross-site scripting)
  * Caminhos completos dos arquivos-fonte relacionados à manifestação do problema
  * A localização do código-fonte afetado (tag/branch/commit ou URL direta)
  * Qualquer configuração especial necessária para reproduzir o problema
  * Instruções passo a passo para reproduzir o problema
  * Prova de conceito ou código de exploit (se possível)
  * O impacto do problema, incluindo como um invasor pode explorá-lo

Essas informações nos ajudarão a avaliar seu relato mais rapidamente.

## Política

Consulte a [Política de Safe Harbor do GitHub](https://docs.github.com/en/site-policy/security-policies/github-bug-bounty-program-legal-safe-harbor#1-safe-harbor-terms)
